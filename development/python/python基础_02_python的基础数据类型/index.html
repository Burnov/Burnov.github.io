2.7、字典dict

2.7.1、数据类型分配

​ 首先了解一下数据类型的分类：

可变数据类型 不可以进行hash计算的数据类型 list、dict、set（集合）
不可变数据类型 可以进行hash计算的数据类型 tuple、bool、int、str（对于字符串的操作都是形成一个新的字符串，而不是改变的原有字符串）

​ 对于字典而言，其key值必须是不可变的数据类型！！！而value则没有要求；

​ 其实，是否可以进行hash计算可以决定其数据的查找速度，对应的数据存储可以根据hash值来存储在对应的内存物理地址上，然后基于物理地址来进行数据查找，这个速度是O(n)的，也是效率最高的！

2.7.2、基本介绍

​ 字典是 key = value 的结构，使用键值对的方法来进行存储的；key必须是不可变的数据类型：tuple、bool、int、str；

​ 字典的优势：key通过hash以后，生成一个hash列表不再基于index索引来进行查找，有了数据的hash表以后，在进行排序就可以提高大大查找效率；

2.9、二进制bytes

2.9.1、初始编码

​ 电报再进行数据传输的时候是通过“.”和“–”，来进行电报传输。

​ 电报器编码格式如下：

​ 同理，再计算机中同样通过高低电平的方式来进行数据传输，也是只有“0”和“1”两个信号；所有的电脑数据传输与存储都是“0”“1”。则我们需要用“0”“1”去代表很多种可能。这就有了第一个二进制转换表格 – ASCII表；

​ ASCII表对饮的是8个bit大小，7个bit就是用来进行英文字母以及其他的特殊字符的。最前面一位是预留位，均为0；

​ 但是这个编码只是英语国家能够使用的编码方式，非英文国家只能去开发自己的编码格式；例如，在我国就有GBK编码（例如GBK 2312），但是每个国家都开发自己的编码格式，再跨国的网站平台进行数据传输的时候就乱套了。所以，就有了Unicode的万国码。

​ 但是Unicode编码使用1个字节来表示英文，但是同时也使用了4个字节来表示中文，中文大约有9万个字，所以，4个字节已经远超9万了，这样则造成了浪费。于是，国际组织奖Unicode升级被升级为了utf-8的编码格式，这样它使用3个字节来表示中文。

​ 通过上面的描述，我们知道了各个编码之间与二进制之间的数据是不能相互识别的，会产生乱码显示情况；
