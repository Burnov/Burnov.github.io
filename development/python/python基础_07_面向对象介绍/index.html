7、面向对象介绍

7.1、面向过程与面向对象

面向过程：

​ 面向过程的程序设计的核心是过程（流水线式思维），过程即解决问题的步骤，面向过程的设计就好比精心设计好一条流水线，考虑周全什么时候处理什么东西。

​ 优点是：极大的降低了写程序的复杂度，只需要顺着要执行的步骤，堆叠代码即可。

​ 缺点是：一套流水线或者流程就是用来解决一个问题，代码牵一发而动全身。

​ 应用场景：一旦完成基本很少改变的场景，著名的例子有Linux內核，git，以及Apache HTTP Server等。

面向对象：

​ 面向对象的程序设计的核心是对象（上帝式思维），要理解对象为何物，必须把自己当成上帝，上帝眼里世间存在的万物皆为对象，不存在的也可以创造出来。面向对象的程序设计好比如来设计西游记，如来要解决的问题是把经书传给东土大唐，如来想了想解决这个问题需要四个人：唐僧，沙和尚，猪八戒，孙悟空，每个人都有各自的特征和技能（这就是对象的概念，特征和技能分别对应对象的属性和方法），然而这并不好玩，于是如来又安排了一群妖魔鬼怪，为了防止师徒四人在取经路上被搞死，又安排了一群神仙保驾护航，这些都是对象。然后取经开始，师徒四人与妖魔鬼怪神仙互相缠斗着直到最后取得真经。如来根本不会管师徒四人按照什么流程去取。

​ 优点是：解决了程序的扩展性。对某一个对象单独修改，会立刻反映到整个体系中，如对游戏中一个人物参数的特征和技能修改都很容易。

​ 缺点是：可控性差，无法向面向过程的程序设计流水线式的可以很精准的预测问题的处理流程与结果，面向对象的程序一旦开始就由对象之间的交互解决问题，即便是上帝也无法预测最终结果。于是我们经常看到一个游戏人某一参数的修改极有可能导致bug的技能出现，一刀砍死3个人，这个游戏就失去平衡。

​ 应用场景：需求经常变化的软件，一般需求的变化都集中在用户层，互联网应用，企业内部软件，游戏等都是面向对象的程序设计大显身手的好地方。

​ 在Python中既可以支持面向过程的代码设计思路，又能够支持面向过程的代码编程思路，是非常灵活的。而这里我们主要学习面向对象的编程思想。所以，要了解面向过程的编程思想，我们就需要掌握一下一些概念：类、对象、实例、实例化等；

​ 继承的简单介绍：

​ 1、继承介绍

​ 与组合相对应，如果类之间有一些关联关系，且两者之间又存在很多相同的属性，这时候去定义类的时候，最好就使用继承的方式来定义。那么，什么是继承呢？这是计算机领域非常重要的概念。它是一种创建新类的方式，在Python中，新建的类可以继承一个或者多个父类，父类又可称为基类或超类，新建的类称为派生类或者子类；（子类，就是基于原有父类派生出来的新的类，它有父类的一些共有属性，又具有自己的特殊属性，这就是派生的概念）

​ 与继承相对应的，我们有一个抽象的概念。抽象就是抽取类似或者说比较像的部分。抽象分为两个层次：

​ 1、将奥巴马和梅西这两个对象比较部分抽取成类；

​ 2、将人、动物比较像的部分抽取成父类；

​ 抽象最主要的作用就是划分类别（可以隔离关注点，降低复杂度）；

​ 所以，抽象是一个“向上”的过程，而继承是一个“向下”的关系；

​ 继承：是基于抽象的结果，通过编程语言去实现它，肯定是经历了抽象这个过程，才能通过继承的方式去抽象的结构。

抽象知识分析和设计的过程中，一个动作或者说一种技巧，通过抽象得到类；

​ 什么是多态：由不同的实例化得到的对象，调用同一个方法，执行逻辑不同；多态指出了对象如何通过他们共同的属性和动作来操作级访问，而不需要考虑他们的具体类是什么。

​ 多态表明了动态（又名–运行时态）绑定的存在，统计重载及运行时类型确定和验证；示例如下：

​ 到此，我们就可以实现多态的第一层特性了。那就是针对不同的实例对象，调用相同的方法，可以获取对应实例对象的返回值；

​ 类似于len()方法，我们可以对str和list等不同的数据类型使用len()方法，从而获取对应的长度；

​ 对于Python而言，其程序本身就是支持多态特性的！如果你仔细思考一下，你会发现，多态其实就是由继承发展而来的，只有继承同一个父类，才会体现出多态的特性。多态就是继承的一种体现方式。

​ 多态实际上是依附于继承的两种含义：“改变”和“扩展”，本身就意味着必须有机制去自动选用你改变/扩展过的版本，故如果无多态，则两种含义就不可能实现。所以，多态和继承是不可分割的，他们是相互关联的！！！

7.3.3、封装

​ 封装是什么呢？抛开面向对象，单独想“装”，就是拿袋子放入东西，什么是“封”么？就是将袋子的口子封上；面向对象中，袋子就是我们的类或者对象，而内部的东西就是数据属性和函数属性。例如：

​ 作为用户看到的只有类名，不知道有额你不的实现逻辑，这就是封装的基本逻辑。但是基于封装，我们还要知道几个基本特性：

​ 1、_ – 单下划线

​ 单下划线开头的属性是属于类内部的隐藏属性（当时这只是一个约定俗成的说法），但是实际上通过 dic 方法可以看到这些属性值，且这些属性值也一样可以被调用。

​ 2、__ – 双下划线

​ 双下线开头的属性也是内部隐藏属性，但是相对于双下线的属性值，其多加了一层修饰“__star”会变成“_People__star”，所以真正在调用的时候，通过“_People__star”来进行调用也是可以的；

​ 3、明确区分内外

​ 在内部就能正常访问参数，而到了外部就“不能”访问参数；

​ 在定义内部数据属性的时候，一定要思考清楚，因为如果一旦随着时间推移，则需要开启其外部访问接口，这是就比较麻烦；

​ 总结：

​ 对于封装，我们最后达到的目的应该是内部可以直接使用该变量，而外部则无法知晓（Python不会进行严格的内部属性值的定义），再提供一个接口供外部来访问该值。如上面的get_star()函数就是一个访问__star变量的一个接口，这就是真正意义上的封装。
