5.1.1、函数的基本介绍

​ 从函数开始，所有的编程不再仅仅值面向过程的编程，为了简化代码，我们可以使用面向函数的编程思想，将功能写入到函数，最后调用函数即可；在这种思想下，我们的代码不会再试简单的顺序执行了，而是基于函数来执行；

5.1.2、返回值的问题

​ 添加返回值，就是将print改为return；

​ 返回值总共有三种情况：

​ 1、没有返回值；

​ 写了return值和不写return值，这种情况，就是正常运行结束，函数运行就结束了；注意不管是写没有写return，只要后面是没有带值，则返回值都是None；

​ 2、一个返回值；

​ 指定的返回值可以是列表、字典、int、str都可以。可以返回任意数据类型，只要返回就可以直接接收；

​ 3、多个返回值；

​ 返回多个值的时候用逗号隔开，且数量必须匹配；

5.2.1、函数的命名空间和作用域

​ 命名空间的定义：

​ 我们知道如果再函数外面定义一个变量，函数内部是可以使用的；但是，在函数里面去定义一个变量，这个变量值在外面是不能够使用的！

​ 在学习函数之前，我们所有的代码都再全局环境下运行的，因此所有的变量名称都属于同一个域环境下（也就是全局名词空间（global namespace）），而函数的定义则新建了一个局部名称空间（local namespace）；

​ 再全局名词空间外，还有一个大的环境，这个由Python解释器提供，我们叫做内置名称空间（built-in namespace）,而所有的内置参数名称都放在这个地方，供全局和局部两个名称空间去调用，这也是为什么我们可以使用tulpe、list、print等默认函数名称的原因；

​ 名称空间的加载顺序：

​ 1、内置名称空间

​ 2、全局名称空间

​ 3、局部名称空间

注意：在局部可以使用全局以及内置的名称空间中定义的名字；但是，在全局只能使用内置名称空间中定义的名字，不能使用局部名称空间中定义的名字；（也就是可以向上找，不能向下找！）

​ set()–set无序排序且不重复，是可变的，有add（），remove（）等方法。既然是可变的，所以它不存在哈希值。基本功能包括关系测试和消除重复元素. 集合对象还支持union(联合), intersection(交集), difference(差集)和sysmmetric difference(对称差集)等数学运算。sets 支持 x in set, len(set),和 for x in set。作为一个无序的集合，sets不记录元素位置或者插入点。因此，sets不支持 indexing, 或其它类序列的操作。

​ frozenset()–frozenset是冻结的集合，它是不可变的，存在哈希值，好处是它可以作为字典的key，也可以作为其它集合的元素。缺点是一旦创建便不能更改，没有add，remove方法。
