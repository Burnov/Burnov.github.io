kubectl命令详解

kubectl的访问方式

基于kubectl的命令，我们都是通过api-server端来进行访问的；

如果不能调用这个端口，者kubectl的管理命令是无法使用的。同时，如果master出现故障，是不影响pod的使用的，但是同时也无法进行管控了；

同时在master的集群环境下，如果修改这个server配置，也是可以调用其他master的节点的api-server接口对k8s集群进行管理的；

注意：k8s的命令是可以通过tab键来补全的，这个类似于shell命令

get

首先，我们可以通过kubectl -h的方式来查看所有的命令；而get则是最常用的资源对象，他是display one or more resourse；显示资源对象的；

这里面关于pod资源的的参数我们简单说一下

  • NAMESPACE – 名称空间的名字
  • NAME – pod的名字，这个是不能重复的，如果重复了之后配置的会覆盖之前的名称
  • READY – 后面一个1表示有多少容器，而前面1表示这个是否在使用
  • STATUS – 表示pod的状态，最好的就是Running了，还有其他状态我们之后再说
  • RESTARTS – 是否重启过
  • AGE – 启动的时长

service的参数

  • NAME – service的名称
  • TYPE – service的类型，ClusterIP和NodePort
  • CLUSTER-IP – 集群IP地址
  • EXTERNAL-IP – 外部地址，这些在云产家里面是需要提供给客户访问的
  • PORT – 端口，如果没有做nodeport转发，则显示就是service的端口；如果是nodeport模式，左侧就是service端口右侧就是宿主机端口；主要使用TCP用UDP少一些；

指定格式以后方便写代码调用里面的内容；
